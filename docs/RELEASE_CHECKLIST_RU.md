# Checklist Публикации Репозитория (RU)

Используйте этот чеклист перед отправкой ссылки преподавателю.

## Обязательное (до отправки ссылки)

1. Проверить `README.md`
- есть итоговый результат (`0.97338` public, дата `2026-02-26`)
- есть разделы "что сработало / что не сработало / идеи"
- есть инструкция воспроизведения

2. Проверить `docs/EXPERIMENTS_RU.md`
- отражены ключевые A/B (особенно `no_color`)
- отражены неудачные/ухудшающие эксперименты
- отражены late-stage результаты и финальный выбор

3. Проверить `docs/PRESENTATION_OUTLINE_RU.md`
- закрывает требования преподавателя
- есть структура слайдов и ключевые цифры

4. Проверить `docs/REPRODUCIBILITY_RU.md`
- понятны входные данные
- понятны launcher'ы
- понятна роль внешних весов

## Обязательное (после загрузки весов на диск)

1. Загрузить большие артефакты на Yandex Disk / Google Drive
- dataset/checkpoint bundles
- ключевые `.pt/.pth` архивы

2. Обновить `weights/README.md`
- вставить реальные ссылки
- добавить размеры
- добавить checksum (желательно `sha256`)

3. Быстро проверить ссылки
- открываются без авторизации преподавателя
- права доступа "по ссылке"

## Желательно (повышает качество релиза)

1. Сделать GitHub Release (`v1.0`)
- приложить краткое описание
- продублировать ссылки на веса

2. Добавить 1-2 скриншота в README
- (в этом репозитории уже есть `assets/`)

3. Добавить post-competition раздел после раскрытия private
- сравнение `public vs private`
- что переоценили / что реально сработало

## Не забыть

- не коммитить датасет и большие веса в git
- не публиковать приватные токены/`kaggle.json`
- не заявлять private-результаты, если они ещё не подтверждены
